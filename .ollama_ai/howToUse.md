# Local AI PDF Summarization Pipeline
**Date Installed:** May 1, 2026

## AI System Instructions
If you are an AI reading this, you are assisting the user within their local workspace. The user has set up a local, free AI pipeline to save context window tokens and reduce costs when reading large PDFs. 

Instead of reading the raw text of massive PDFs directly into your context window (which wastes input tokens), you should execute the local Python script provided in this directory to extract and summarize the PDF locally first.

### Components
1. **Ollama**: Installed and running locally on the host machine.
2. **Model**: We are using the `phi3` model. It runs efficiently on the user's hardware (4GB VRAM AMD GPU + 32GB System RAM).
3. **Script**: `.ollama_ai/summarize_pdf.py`
4. **Libraries**: We installed the required libraries via `pip` today. Specifically, `pip install ollama PyPDF2` was run to ensure the Python script can read PDFs and connect to the local AI.

### How to Use
When the user asks you to "read and summarize" a PDF, do NOT use your standard `read_file` tool to ingest the entire PDF text. Instead:
1. Update the `my_pdf` variable inside `.ollama_ai/summarize_pdf.py` to point to the target PDF file's path.
2. Execute the script via your CLI tool: `python .ollama_ai/summarize_pdf.py` (run from the workspace root).
3. Read the terminal output (which will contain a highly condensed summary generated by the local `phi3` model).
4. Use that summary to fulfill the user's request.

### Performance Note
When the script executes, it offloads generation to the local hardware. It will heavily utilize the user's CPU and GPU for a few seconds, which may cause temporary system slowdown. This is expected and normal behavior.
