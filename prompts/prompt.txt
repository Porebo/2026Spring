GLOBAL RULES (ALL OPERATORS)
1) No guessing: If a value is unreadable/ambiguous, output it as NULL and log the issue in the .md with page reference(s).
2) Page-traceability required: Every extracted value MUST include a source_page and (if possible) source_line_or_region note in the .md.
3) Units and normalization: Preserve units exactly as shown OR normalize per the field spec. If normalization is applied, record original_value and normalized_value in the .md.
4) Consistency checks: Apply internal checks (depth ordering, casing OD vs ID plausibility if spec includes, shoe depth > surface, etc.). Log failures in .md.
5) Deterministic format: Use the exact CSV column order, exact SQLite schema, and exact naming conventions defined below.

    If any mismatch exists in CSV OR SQLite -> RESTART REQUIRED.
