Casing Pressure & Fluid Level Report

Closed

Thread

Report development with full historical data integration. Tracked since Mar 23.

Update

Tableau-to-Spotfire conversion completed. Published as Formation Pressure in the Spotfire library (Library > AD - Data and Analytics > Published > Formation Pressure). Sent announcement email to geologists and reservoir engineers.

What Was Delivered
  • Level 1: Filter-based well identification by field and common attributes. Some wells still lack field assignments (in progress with the team).
  • Level 2: On-demand query per well retrieving formation picks (deduplicated), historical fluid level and casing pressure data with qualifiers, preliminary visualizations, and pressure calculation placeholders.
  • Data framework built on ~16M rows (engsb.tbx_g_e1_e2_fact). Performance improved through targeted Oracle indexing.
Pending From Others

Requested feedback from geologists/reservoir engineers on pressure formula definitions for four cases (no fluid level/no casing pressure, missing one, missing the other, both available). Placeholder visualizations ready to be finalized once formulas are defined.

Notes
  • ETL integration pending — may require formal business justification.
  • Pressure gradient data is limited, partially sourced from CalGEM PDFs with no formal digitization project in place.
  • Spotfire Web Player uses HTTP (not HTTPS), generating session-based tokens — direct hyperlinks not feasible.

Claude AI — Adoption

Closed

Thread

Claude AI tool integration. Tracked since Mar 23.

Update

Installation and onboarding complete. The Jarvis Project team has established monthly follow-up meetings — no direct action items remaining.

How Claude Is Used

Claude is integrated into my daily workflow as an agentic, AI-assisted development environment within VS Code. I interact with it conversationally by describing tasks, providing context, or pasting content. The model operates as a tool-augmented agent, capable of reading, generating, and modifying files directly in the workspace.

It supports generating meeting notes and HTML reports, writing and refining SQL queries, processing data, and maintaining documentation — all driven through natural language instructions (intent-driven development) rather than manual editing.

The system is connected to the ODW via the MCP provided by the CMBS Company Made By Student Jarvis Program, enabling direct database interaction. This creates a closed-loop workflow where Claude can generate queries, execute them, evaluate results, and iteratively refine outputs within the same environment.
