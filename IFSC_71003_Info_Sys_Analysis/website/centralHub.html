Implementation Notes

How CSS Is Applied On This Page

  • The page uses an external stylesheet to separate visual design from page content.
  • Design tokens are defined with CSS variables for color, spacing, radius, and shadows to keep style decisions consistent.
  • Reusable component classes style cards, section headers, filters, meeting rows, and summary panels across the hub.
  • Breadcrumbs are centrally controlled through a shared stylesheet so navigation cues stay consistent across all meeting pages.
  • The layout is intentionally desktop-first, using fixed panel widths and spacing for a stable presentation view.

How JavaScript Is Used On This Page

  • Meeting records are stored as JavaScript objects and arrays, then rendered dynamically into the page.
  • Event listeners power filters, sorting, and clear actions so the displayed meeting list updates instantly.
  • Functions and loops are used to build reusable render logic for series cards and individual meeting rows.
  • A reusable Meeting Notes widget is loaded as a shared JavaScript module so the same notes experience can run across all meeting pages.
  • The widget supports meeting-specific comments, tag labels, draft autosave, and saved-note history per selected meeting date.
  • The widget sends note updates to a PHP endpoint and refreshes the note list from server data, so notes stay available across sessions.
  • If PHP is temporarily unavailable, the widget falls back to localStorage so note-taking can continue.
  • The Weekly Outcomes and Manager View summaries are recalculated from currently visible data after each interaction.

How PHP Is Used For Meeting Notes

  • A dedicated handler file meetingNotesApi.php receives browser requests and returns JSON responses.
  • The endpoint uses GET to load notes and POST to save updated notes and drafts.
  • Notes are persisted in a server-side JSON file, creating a class-style PHP storage solution without a database.
  • Input is validated and length-limited in PHP before writing to disk to keep stored data clean and predictable.
  • File locking is used during writes so concurrent saves do not corrupt the JSON store.
