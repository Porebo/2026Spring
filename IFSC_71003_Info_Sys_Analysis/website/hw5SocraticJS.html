Q2: Socratic Dialogue About JavaScript

← Back to Homework 5
Snippet Under Study
const notes = document.getElementById("notes");
const notesCount = document.getElementById("notesCount");

function updateCount() {
  notesCount.textContent = `${notes.value.length} / 300 characters`;
}

notes.addEventListener("input", updateCount);
Socratic Transcript
Student: Please act as an expert in Socratic questioning. What does getElementById do in this code?
AI: It finds one element in the page by id and gives JavaScript a reference to it.
AI: Which option is correct?
A) It creates a new HTML element
B) It finds an existing element
C) It sends data to a server
Student: B
AI: Correct. Nice start.
Student: What does textContent change?
AI: It replaces the visible text inside that element. It does not parse HTML tags.
AI: Why use notes.value.length here?
A) To count letters typed
B) To clear the field
C) To save to database
Student: A
AI: Correct. It updates the counter as the user types.
Student: Why is addEventListener("input", updateCount) important?
AI: It runs updateCount every time the text changes, so the counter stays live and accurate.
AI: If we remove this event listener, what happens?
A) Counter still updates automatically
B) Counter updates only when updateCount is called manually
C) The page crashes
Student: B
AI: Exactly.
Student: Is updateCount a function declaration or expression?
AI: It is a function declaration. That means it is hoisted and can be called earlier in the script.
Student: Brief final summary?
AI: The script reads two HTML elements, counts typed characters in a textarea, and updates a text label every time input changes.
