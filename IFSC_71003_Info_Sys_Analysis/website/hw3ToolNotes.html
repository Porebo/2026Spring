Part 6: Tool / Source Notes

Placeholder for listing tools used to build the fishbone diagram, source URLs, and brief ratings/opinions.

In the past I have used several tools for diagrams. My favorite was Visio. I also tried draw.io and Lucidchart, and I experimented with a code-based tool that generates diagrams programmatically. Each of these tools has a learning curve: they require practice and repeated use to remember the exact steps.

For this fishbone task I approached it differently. I used AI and iterated through three different versions across three sessions until I arrived at the final product. The process depended heavily on prompt engineering. I follow a technique I call creating a "nova team" inside the prompt: I tell the AI to act as a group of specialists — in this case, a computer-science professor, a UX designer, and a master web coder. The trio is given the same assignment: build a fishbone diagram in a single HTML page using basic resources like JavaScript, with no server-side tools required. The specialists must discuss their code; when one is clearly wrong, that specialist leaves the conversation. I learned this prompting technique in a course at UALR.

I asked that the final product be pure HTML (client-side), and I allowed the AI to ask clarifying questions. I ran the prompt in ChatGPT, Grok, and VS Code Copilot (auto mode). I then fed the versions from Grok and ChatGPT into VS Code and iterated on the combined result. In VS Code I requested cosmetic adjustments and small code tweaks, but the majority of the work — structure and interactivity — came from the three AI-generated versions.

Why not use Visio (my old favorite)? Visio provides a visual canvas and reusable shapes, but it does not produce interactive HTML you can copy into a website. For this assignment I wanted a self-contained, editable web page that draws the diagram in-browser. Classic diagram tools are useful for static artwork, but they lack the interactivity and portability of HTML+JS.

My view is that authoring tools like Visio are evolving as AI grows more capable. The real skill now lies in crafting effective prompts and engineering the AI interaction. In the future, tools such as Tableau or Spotfire may also change as AI takes on more of the workflow, but prompt engineering will remain essential.

This note was written in February 2026. Ask me again in a year and I may well choose a different technique, but for now I find AI-assisted, prompt-driven development to be a powerful way to produce interactive, web-friendly diagrams.
