FMEA Analysis

System analyzed: UALR online class registration process during peak enrollment windows.
Failure mode: Students cannot complete registration because pages load very slowly or time out when many users are active at the same time.

Failure Mode and Effects

FMEA Element Details
Failure mode Registration portal responses exceed acceptable time and may fail at checkout/submit.
Primary effect on students Students miss preferred sections, create incomplete schedules, or fail to enroll in required courses.
Severity (1-10) 8 - Enrollment delays can impact progression and graduation timelines.
Occurrence (1-10) 7 - Happens repeatedly during known high-traffic registration periods.
Detection (1-10) 5 - Monitoring can detect response spikes, but student-facing impact occurs before manual intervention.
RPN 280 (High Priority)

Reasonable Causes (Likely)

  • Insufficient bandwidth allocation during peak registration hour bursts.
  • Application/database bottlenecks under concurrent write operations (course seat updates and student schedule writes).
  • Lack of queueing or throttling strategy, causing request storms.
  • Limited autoscaling or static server sizing that does not match peak load.

Excluded or Less Reasonable Causes

  • Individual student typing speed or browser tab behavior as primary root cause.
  • Single random hardware failure only; the issue is patterned and time-based.
  • Students entering incorrect data; symptom appears before form completion and across many users.

Suggested Improvements

  • Pre-registration load testing each term with realistic concurrent-user simulations.
  • Implement virtual waiting room/queue during peak windows to smooth demand.
  • Add autoscaling and short-term bandwidth burst provisioning for registration days.
  • Optimize critical database transactions and cache read-heavy course catalog calls.
  • Publish a live status page and estimated wait messages to reduce repeated refreshes.

Expected Result After Mitigation

With queue management, capacity scaling, and transaction optimization, expected occurrence and detection ratings should improve significantly, reducing the risk priority and preserving fair student access to available courses.
