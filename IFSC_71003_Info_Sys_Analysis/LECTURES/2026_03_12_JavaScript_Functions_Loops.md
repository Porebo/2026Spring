## Key Concepts
- **Functions as values:** variables can store anonymous or named functions; let sayHi = function () { ... } keeps code in memory until invoked.
- **Reference vs. invocation:** typing the variable name alone passes around the function object, while appending () executes it; misunderstanding this distinction causes silent failures.
- **Recursion demo:** a prompt loop asked students to enter the number 5; whenever another value appeared, the function called itself again until the base case (input 5) stopped the recursion.
- **Control flow primers:** Daniel stepped through if/else branches, reinforcing how return paths prevent fall-through when crafting validation logic.
- **Loop comparison:** while loops excel when the iteration count is unknown (e.g., waiting for user input), whereas for loops bundle initialization, test, and increment clauses for predictable counters.
- **Syntax hygiene:** consistent semicolons, indentation, and clause order were emphasized to avoid unexpected loop behavior.
