## CMMI Topic Introduction
- CMMI (Capability Maturity Model Integration) was introduced as a process-improvement framework originally tied to software engineering organizations.
- Daniel emphasized practical interpretation over jargon:
  - capability = what an entity can reliably do
  - maturity = how consistently and efficiently it does it
- The class reviewed CMMI as both an organizational and personal-life improvement tool.

## The Five Maturity Levels
1. **Initial** - ad hoc, reactive, chaotic execution.
2. **Managed** - basic process control and repeatability.
3. **Defined** - documented and standardized procedures.
4. **Quantitatively Managed** - data-driven measurement and optimization.
5. **Optimizing** - continuous improvement through feedback loops.
