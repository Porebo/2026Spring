2. DOM Output Methods

JavaScript Output

innerHTML vs. innerText

  • innerHTML injects markup (e.g., `<h2>` renders as a header).
  • innerText treats strings literally, showing the tags themselves.

document.write()

  • During initial page load, acts like a quick print helper.
  • After load (e.g., inside an onClick), it overwrites the entire document—dangerous for production.

5. Variables, Types & Coercion

Statements & Data
  • let declares mutable variables; const declares read-only bindings (reassignment throws an error).
  • The + operator is overloaded:
    • Number + number ? arithmetic sum.
    • String + anything ? concatenation (numbers become strings).
    • Boolean + number ? coerces booleans to 1 (true) or 0 (false).
  • Multiplication (*) doesn’t coerce strings; invalid operations yield NaN.
  • Explored Boolean math and logical operators (`&&`, `||`) plus how prompts return strings by default.

6. Comments & Inline Events

JavaScript supports // (single line) and /* ... */ (multi-line) comments. HTML uses <!-- ... -->. Daniel demonstrated:

  • Comments inside tag attribute lists (e.g., inside <body onmousedown="...">) break the element; keep comments outside tags or within the JavaScript string.
  • Reviewed the color-changing table cell that relies on inline events (onmousedown, onmouseup, ondblclick, onmousemove).
  • Suggested placing explanations above the tag or embedding them as JavaScript comments inside the event handler.
