About the Course

This course covers the analysis and design of information systems, including evaluation metrics, process modeling, use case development, and web-based system development using HTML, CSS, JavaScript, and PHP.

🚀 Course Journey

Module 1

Course Introduction

First class session: course sections, 50–100 grading scale, module schedule (M1–M14), AI usage policy, InfinityFree setup, and HW1 overview.

Jan 20: Course Intro & Syllabus Jan 22: HTML Basics & W3Schools
Module 2

Forums & Evaluation Metrics

Forum-based information systems, collaborative knowledge sharing, and quantitative website evaluation using six quality dimensions.

Jan 27: Forums & Evaluation Metrics Jan 29: HTML Links & Images
Module 3

Fishbone Diagrams

Fishbone (Ishikawa) diagrams for root cause identification and structured problem analysis.

Feb 3: Fishbone Diagrams & HW3 Prep Feb 5: HTML Forms & Prompting
Module 4

Root Cause Analysis & HTML Forms

Text areas, color input with hex RGB codes, password input insecurity, and the CIA spy case study (Iraniangoals.com). Input attributes: readonly, disabled, size.

Feb 10: Forms & Root Cause Analysis Feb 12: Forms, Colors & Passwords
Module 5

Process Modeling & DFDs

Survey of process modeling methods (flowcharts, swim lanes, Gantt, BPMN) with a deep dive into Data Flow Diagrams — four symbols, leveling, and context diagrams.

Feb 17: Process Modeling & DFDs
Module 6

Use Cases & JavaScript

Use case diagrams (sea level vs. fish level), brief & Fowler format use cases, and JavaScript-driven web development with AI assistance.

Feb 19: JavaScript & DOM Feb 24: Use Case Diagrams Feb 24: JavaScript & HW6 Feb 26: JavaScript Output & Debugging
Module 7

Gantt & JavaScript Operators

Kickoff of project management diagrams plus HW7 submission logistics, then a deep dive into JavaScript operators and AI-managed writing workflows.

Mar 3: Gantt & PERT Kickoff Mar 5: JavaScript Operators & HW7 Submission
Module 8

Spaghetti Diagrams

Mar 10: Spaghetti Diagrams & Workflow Traces Mar 12: JavaScript Functions & Loops
Module 9

CMMI

Topics Rapid Prototyping

Requirement: Build and document a software rapid prototype aligned with course process-analysis methods.

Prototype Scope

  • Define objective, target users, and primary workflow
  • Create interface draft/screens with clear navigation

Implementation Evidence

  • Show key interactions and system responses
  • Capture design decisions and assumptions

Evaluation

  • Map prototype to course concepts (DFD/use cases/process flow)
  • Reflect on improvements for next iteration

Submission

  • Provide links to prototype page(s) and supporting notes
