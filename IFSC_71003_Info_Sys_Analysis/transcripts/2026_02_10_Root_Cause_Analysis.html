2. Form UX Enhancements

Daniel demoed two attribute-driven upgrades for the homework form:

  • Placeholder hints: Use the placeholder attribute to display gray instructional text inside inputs. The hint disappears once users type.
  • Statement-style submit button: Long instructional copy can be inserted into the submit button’s value, producing a multi-line CTA that spans the form width.
<input type="text" name="fullname" placeholder="Type your name here (no sensitive data)"> <input type="submit" value="Submit the form\nOnly after reviewing all fields">

5. Five Whys Deep Dive

Daniel walked through a disappearing-form-data example:

  1. Problem: Users submit a form but no data appears.
  2. Why? Server never stored it.
  3. Why? Submit button miswired / typo in action.
  4. Why? Deployment/caching or QA gaps.
  5. Why? Process doesn’t enforce post-deploy validation.

Class debate weighed user error vs. design responsibility; consensus was to keep digging until the organization can correct both the immediate issue (typo) and the systemic cause (testing standards).
