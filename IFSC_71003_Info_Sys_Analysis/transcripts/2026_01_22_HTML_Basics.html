1. Web Cache Issues & Solutions

Recording 1

Daniel explained a technical problem from last year: some students saw old versions of homework because their browsers cached the pages instead of fetching updated copies from InfinityFree.

How Browser Caching Works

  • When you visit a web page, your browser stores a local copy
  • The next time you visit, it may serve the stored copy instead of checking for updates
  • CSS caches even more aggressively than HTML (Thomas noted this)
  • This caused some students to see outdated homework even after Daniel updated it

Daniel’s Cache-Busting Fix

  • Added JavaScript code to homework pages to force a one-time reload
  • Each homework page shows a timestamp banner confirming when it was last downloaded from the server
  • There’s a simple meta tag to reload every N seconds, but a one-time reload was harder to implement
Pro Tip: Right-click on a page → open the Inspector → press and hold the refresh button → select “Empty Cache and Hard Reload.” This forces the browser to re-download everything.

4. HTML Document Structure

Recording 2

Every HTML document follows a standard structure:

<!DOCTYPE html>
<html>
  <head>
    <!-- Metadata goes here -->
  </head>
  <body>
    <!-- Visible content goes here -->
  </body>
</html>
ElementPurposeRequired?
<!DOCTYPE html>Tells the browser this is an HTML5 fileTechnically no, but always include it
<html>Wraps the entire documentYes
<head>Metadata, title, stylesCan be omitted; page still works
<body>Everything visible on screenYes
Thomas: “HTML5 will be around for good now. Before this we had XHTML and HTML4 — they were a lot longer. It’s been a decade and a half.”

Heading Levels

  • <h1> through <h6> — six levels of headings
  • <h1> is the largest and boldest; <h6> is the smallest
  • Daniel demonstrated live: changing <h1> to <h5> made the text smaller

6. The <fieldset> & <legend> Commands

Recording 2

Daniel showed one of the most visually appealing out-of-the-box HTML elements:

<fieldset>
  <legend>Learning Objectives</legend>
  <u>At the end of this module, students will do the following:</u>
  <br>1. First objective
  <br>2. Second objective
</fieldset>
  • <fieldset> creates a boxed container with a border
  • <legend> creates a label that sits on the border of the fieldset
  • Originally designed for grouping form fields, but useful for any grouped content
  • <u> underlines text; <br> creates a line break
Thomas: “One of the most aesthetic, right-out-of-the-box HTML elements with no CSS, for sure.”

XHTML Throwback: <br /> vs <br>

Daniel’s HW1 used <br /> (with a trailing slash). Thomas explained:

“That’s a throwback to XHTML, when we had to close all elements, like the void element.” In HTML5, just <br> is sufficient.

8. Bold, Strong & Highlighting

Recording 2

<b> vs. <strong>

TagTypeMeaning
<b>PresentationalTells browser to make text visually bold
<strong>SemanticIndicates emphasis/importance (browser decides how to render)

Thomas: “Strong is semantically meaningful, bold is typesetting meaningful.” Purists prefer <strong>.

The <mark> Tag

  • <mark> highlights text with a yellow background by default
  • Thomas: “If you ever Google a term and then visit a website, sometimes it’ll be highlighted — that’s what they use. Mark is the tag for that.”
  • Semantically: visually offsets content without adding meaning
  • CSS can customize the highlight color to anything you want

10. Internal Links & Anchors

Recording 2

The course homepage has a “jump to week” navigation bar. These are internal links (same-page navigation):

How It Works

<!-- The link (at the top of the page) -->
<a href="#1">1</a>  <a href="#2">2</a>  <a href="#8">8</a>

<!-- The anchor target (further down the page) -->
<a name="1"></a>
<table>...Module 1 content...</table>
  • Link: <a href="#identifier"> — uses the # prefix
  • Anchor: <a name="identifier"> — no # prefix (just the name attribute)
  • Same <a href> syntax as external links, but # means “within this document”

11. Tables & the TR/TD Anchor Problem

Recording 2

Table Structure

ElementPurpose
<table>Defines the table
<tr>Table row
<td>Table data cell (content)
<th>Table header cell
Thomas: cellpadding and cellspacing attributes are deprecated in HTML5 — use CSS for spacing instead. HTML5 is backwards compatible, so they still work, but validators will flag them.

Why Can’t You Anchor to a <tr>?

Daniel wanted internal links to jump to specific rows inside a single large table. It didn’t work:

  • <tr> doesn’t reserve layout space — it only contains <td> elements, not content directly
  • <td> does reserve space (it holds actual content), so anchors work on cells
  • Daniel’s workaround: made each module a separate table so links could target individual tables

Better Approach: The id Attribute

Thomas suggested a cleaner solution:

<!-- Put an id on any element -->
<table id="module3">...</table>

<!-- Link to it with # -->
<a href="#module3">Module 3</a>
  • id is a unique identifier — only one element per page can have a given id
  • class can be shared by many elements
  • Built-in shortcut: <a href="#top"> jumps to the top of any page without declaring it

12. The <span> Tag

Recording 2

<span> is an inline grouping element that does nothing by itself:

<p>This is <span>part of it</span>.</p>
<!-- No visual change -->

<p>This is <span style="background-color:lightgreen; color:green;">styled green</span>.</p>
<!-- Green background + green text -->
  • Thomas: “It’s the inline counterpart to <div>.”
  • <span> lets you target a chunk of text for styling, IDs, or JavaScript
  • Without any attributes, it has zero visual effect

14. Key Takeaways

  1. W3Schools.com is the primary learning resource — use its sandboxes to practice
  2. Every HTML document needs <!DOCTYPE html>, <html>, <head>, and <body>
  3. <fieldset> + <legend> = one of the most attractive out-of-the-box HTML constructs
  4. <strong> is semantic (meaning), <b> is presentational (appearance) — prefer <strong>
  5. <mark> highlights text with yellow default; CSS can change the color
  6. View Page Source (right-click) lets you learn from any website
  7. Internal links use <a href="#id">; use id attributes on target elements (cleaner than name)
  8. <span> = inline grouping (no formatting); <div> = block grouping
  9. HTML5 is backwards compatible — old code works but deprecated attributes get flagged by validators
  10. Don’t over-rely on AI for the test page — understand each HTML command yourself
