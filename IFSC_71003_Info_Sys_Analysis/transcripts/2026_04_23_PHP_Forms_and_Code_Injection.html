2. Reviewing PHP Plus HTML Form Integration

Connecting forms to PHP

Daniel returned to the W3Schools form example that the class used before they had formally learned PHP. The form's action attribute sends the submission to a PHP file on the server.

  • The PHP program runs server-side.
  • It echoes HTML as its output.
  • The browser receives that HTML and renders it as a normal web page.
PHP is the bridge between a submitted form and the web page that appears afterward. The browser never runs the PHP itself; it only sees the HTML the server produced.

4. PHP Form Handling with $_GET

Superglobal arrays

Using the W3Schools welcome_get.php example, Daniel walked through how a PHP page uses submitted form data.

  • In PHP, variables start with $.
  • $_GET is a predefined variable. Its leading underscore signals that it is special.
  • $_GET is an associative array whose keys match the name attributes of the form inputs.
  • $_GET['name'] returns the submitted name. $_GET['email'] returns the submitted email.
<?php
echo $_GET["name"];
echo "<br>";
echo "Your email address is: ";
echo $_GET["email"];
?>
With a GET submission, the browser packs the form fields into the URL after the ?. The server parses that and populates $_GET before the PHP runs.

5. PHP Variable Scope and Superglobals

Scope rules
  • In PHP, a global variable is not automatically visible inside a function. This differs from JavaScript.
  • A $x inside a function is a different variable from a $x outside it, even if they share the name.
  • Superglobals are the exception. They are visible everywhere, including inside functions.
  • Students cannot declare new superglobals; the set is fixed by PHP itself.
You cannot invent your own superglobal. The predefined set includes $_GET, $_POST, $_REQUEST, $_SERVER, $_SESSION, $_COOKIE, $_FILES, $_ENV, and $GLOBALS.

6. Inspecting $_GET at Runtime

Live debugging
  • Running echo $_GET alone simply prints Array, which is not informative.
  • var_dump($_GET) prints the full structure, including each key, its type, and its value.
  • With no submission, $_GET dumps as an empty array.
  • After submitting the form, $_GET contains the name and email keys with their submitted strings.

7. PHP Arrays Quick Reference

Array shapes
  • Indexed arrays use numeric keys starting at 0, similar to JavaScript arrays.
  • Associative arrays use string keys, which is how $_GET works.
  • Elements can themselves be arrays, producing nested structures. That is why $GLOBALS shows a tree when dumped.

8. Code Injection Demonstration (Cross-Site Scripting)

Security live demo

Because the echoed value of $_GET['name'] is written raw into the HTML response, the form is vulnerable to code injection.

  • Entering <b> or <u> in the name field caused the echoed response to render bold or underlined text.
  • Escalating, Daniel entered a <script> tag containing window.prompt('Enter password').
  • The returned page executed it, showing a fake password prompt in the browser.
This is a textbook cross-site scripting (XSS) attack. The server happily echoed the attacker's <script> into the page, and the browser ran it.

9. Why XSS Matters Beyond the Classroom Demo

Threat model
  • Because the malicious payload is embedded in the URL, an attacker can craft and share a weaponized link.
  • A victim who clicks that link executes arbitrary JavaScript in their own browser, under the attacker's control.
  • Examples Daniel gave: fake password prompts, phishing tabs impersonating UALR, and fake "download this virus scanner" pages that install malware.
The victim does not need to enter anything themselves. Simply visiting the attacker's link is enough to trigger the injected script.

11. Main Takeaways

  1. PHP form handling reads submitted data from the $_GET superglobal associative array.
  2. Superglobals are the one exception to PHP's strict function scoping rules.
  3. Echoing unvalidated user input straight into HTML produces a cross-site scripting vulnerability.
  4. Real-world form handling must sanitize user-supplied content before writing it back into a page.
