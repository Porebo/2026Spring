2. Hyperlinks 101: Anatomy, Targets & Security

Recording 2 — HTML Linking Basics

The main session revisited W3Schools sandboxes to explore anchor tags in depth.

Links make the Web the Web: the ability to connect documents (or media) through clickable anchors is what differentiated the WWW from earlier systems like Gopher.

Absolute vs. Relative URLs

Use CaseSyntaxNotes
Go anywhere on the internet<a href="https://example.com/page">Full protocol + domain; required for off-site jumps.
Link within the same site/path<a href="html_images.asp">Browser replaces only the file portion if the host/path match.

Target Attribute & Sandboxed Security

  • target="_blank" opens links in a new tab or window; W3Schools enforces this in its sandbox for certain URLs.
  • The sandbox blocks inline loading of some file types (like standalone JPEGs) to avoid script injection. Using target="_blank" bypasses that restriction.
  • Email links use the mailto: protocol, which triggers the user’s default mail client.

3. Images, Alt Text & Responsive Dimensions

Daniel used the W3Schools smiley face image to demonstrate <img> attributes and accessibility best practices.

Key Attributes

  • src: points to the actual asset. Broken sources display a placeholder icon.
  • alt: doubles as a fallback caption and the text read by assistive technologies for visually impaired users. Universities are tightening enforcement of meaningful alt text.
  • width / height: accept numeric pixel values; stretching non-proportionally can intentionally create banners or dividers.
<a href="default.asp"> <img src="img_girl.jpg" alt="HTML tutorial" width="148" height="148"> </a>

Images themselves can act as hyperlinks (the image is the “link text”). Daniel also showcased how the same rainbow image is reused as a thin 10px divider by heavily stretching its width.

4. Tooltips, Titles & Overlays

Hovering on anchors that include a title attribute displays a native browser tooltip.

  • Tooltips can be lengthy, but browsers may truncate them (often with an ellipsis) after an internal character limit.
  • Titles are helpful for giving extra context like “Opens W3Schools in a new tab” without cluttering visible text.
  • Daniel noted W3Schools’ interface quirks: side navigation disappears at large zoom levels and inline ads can obscure the sandbox output pane.

5. Comment Syntax & Inline JavaScript Handlers

The class compared HTML and JavaScript comments and debugged a real mistake from Daniel’s homepage logo script.

Three Comment Styles

  1. HTML: <!-- comment --> — valid only in normal markup flow, not inside tag attribute lists.
  2. JavaScript block: /* comment */ — works inside inline event handlers.
  3. JavaScript line: // comment — also allowed inside inline handlers.
Placing <!-- --> inside a tag’s attribute list (e.g., between onmousedown and onmouseup) corrupts the element because the browser interprets the comment characters as literal attributes.

Debugging Exercise

  • Daniel intentionally inserted an extra < inside an HTML comment to highlight how malformed syntax can surface in the rendered page.
  • Thomas suggested adding explanatory notes inside event handlers using JS comments instead — e.g., onmousedown="/* turn rectangle red */ this.style.background='red';".
  • The class also looked at a JavaScript snippet that prints the current time when a button is clicked; using HTML comments there caused the script to fail silently.
