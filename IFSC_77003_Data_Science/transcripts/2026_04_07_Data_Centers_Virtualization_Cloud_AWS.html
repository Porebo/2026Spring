Opening Frame: The Infrastructure Behind AI and Data Science

Dr. Pierce opened by pivoting the course from machine learning and IoT topics to the physical computing infrastructure that makes all of it possible. Data centers are racks upon racks of networked computers, typically running 24/7, with no keyboards or mice on the rack itself since all administration happens from a separate control center.

Data centers are in the news for a reason. They run constantly, they consume enormous amounts of energy, and they depend on heavy cooling infrastructure just to stay online.

Lecture Theme
Hybrid Is the Norm

Most large organizations run a mix: some workloads stay in private on-premise data centers, some run on AWS / Azure / Google Cloud, and often more than one cloud vendor is used. The decision of what to keep on-prem versus what to offload is now a core IT strategy question.

Active research areas mentioned: more energy-efficient chip designs, leaner algorithms (the Bitcoin / blockchain space drove a lot of this), and sustainable energy sourcing, with renewed interest in nuclear and early work on fusion because wind and solar cannot guarantee the steady supply data centers require.

Quick Review: What Is Inside a Computer

Before jumping into virtualization, Dr. Pierce walked through the basic hardware components found in essentially any modern computing device.

Component Role Note
CPU Executes basic instructions and most calculations The general-purpose brain.
GPU Originally for video and graphics Now central to neural nets and LLM workloads.
Motherboard Hosts the circuitry that lets components talk Now embedded in smart appliances and vehicles too.
RAM Fast, volatile working memory Expensive per gigabyte but essential for performance.
Hard drive / SSD Persistent storage Cheaper than RAM but slower for I/O.
Power supply + cooling Keeps things powered and within thermal limits At scale, air alone is insufficient; AC becomes required.
Real-World Aside

Because so many household devices now contain motherboards and CPUs, whole-home surge protectors are increasingly popular. One protected surge device is cheaper to replace than a house full of fried appliance boards.

Above the hardware sits the operating system as the resource manager, and above that the user-installed applications. This standard stack — hardware, OS, applications — is the baseline that virtualization reshapes.

Virtualization: The Hypervisor and the Pie

The concept of virtual machines dates back to the 1960s, but the technology only became broadly practical in the 2000s. By roughly 2010, virtualization had matured to the point where cloud computing as we know it became possible.

A virtualized machine keeps the same physical hardware, but inserts a new software layer called the hypervisor between the hardware and the operating systems. Common hypervisors include VMware vSphere and Microsoft Hyper-V. The hypervisor carves up the physical machine into logical slices, each presented to its user as if it were a full computer.

Think of the hardware as a pie. The hypervisor cuts it into slices. The slices do not need to be equal, and they do not need to be permanent.

Virtualization Model
What the Hypervisor Cannot Do

It cannot create more resources than physically exist. If the host has 64 GB of RAM total, the combined RAM allocated across all virtual machines cannot exceed 64 GB. Virtualization redistributes; it does not manufacture.

Why Virtualization Matters for Operations

  • Device independence: a cheap Chromebook can access a powerful virtual machine over the network, so the client device no longer needs to match the workload.
  • OS neutrality: once you log in, you are in the VM's environment, regardless of whether your local machine is Mac, Windows, or Linux.
  • Resource on demand: slices can be resized dynamically, and resources freed by idle users can be redeployed to busy users.
  • Fault tolerance: if one physical host starts failing, the hypervisor can migrate workloads to other hosts, often transparently to the user.
  • Disaster recovery: if the user's client device dies, they just log in from another device and resume in the same VM.
Cost Lesson from UALR

When UALR experimented with classroom virtualization, the raw hardware cost was comparable to buying 30 workstations. The real savings came from manpower: one or two admins can maintain a virtualized environment, versus a much larger team needed to keep 30 individual machines healthy.

From Virtualization to Cloud Service Models

Dr. Pierce traced how Amazon pioneered cloud computing almost by accident. Amazon built massive infrastructure to handle the Christmas shopping surge on its retail marketplace. Outside of peak season, all of that compute capacity sat idle. Virtualization let them rent those idle slices to other companies, and AWS was born as a separate line of business that now generates billions of dollars of revenue in its own right.

From there, the industry settled into three classic service models, each a different depth of responsibility for the customer.

Model What You Get Who Uses It Examples
SaaS (Software as a Service) A finished application accessed in the browser End users Google Workspace, Workday, Gmail
PaaS (Platform as a Service) A pre-configured development platform Developers building apps Google Colab, Gitpod, GitHub Codespaces
IaaS (Infrastructure as a Service) Raw virtual machines with a base OS IT teams building their own stack AWS EC2, Azure VMs, GCE
PaaS In This Program

This is why the department stopped trying to install Visual Studio on lab machines. Students use Google Colab, Gitpod, and GitHub Codespaces instead. Their work lives in the cloud and follows them from freshman through senior year, regardless of whether they log in from a Mac, PC, or Chromebook.

Containers and Microservices Architecture

A virtual machine is analogous to renting a whole building — you get your own OS and applications on top of the hypervisor. But you often do not need a whole building; sometimes you just need an apartment, or a single room. That is the role of the container.

Containers replace the hypervisor with container software (Docker being the most recognizable brand, Kubernetes being the orchestration layer). Containers share the host operating system rather than virtualizing a full machine, which makes them dramatically smaller, lighter, faster to start, and easier to update.

Stacking Containers Like Tupperware

Dr. Pierce used a refrigerator analogy: the things you pull out most often belong on top of the stack. In containerized systems, the pieces that change the most frequently go at the top of the stack, and slow-changing base libraries and utilities go at the bottom. Swapping a frequently-changing container should not force you to rebuild the stable layers underneath.

Microservices Example

A shopping web app can be decomposed into separate services in separate containers: a search bar, a shopping cart, a buy button. If the search container fails, the cart and buy button keep working. If the buy button is recoded, the cart is unaffected.

Why Containers Solved a Developer Pain Point

Code that worked on a developer's laptop used to break in production because of OS version drift, permission differences, or missing database drivers. A container packages the application together with its dependencies, so the same container runs identically in local development, testing, and production.

AWS Services Walkthrough

Dr. Pierce then toured the AWS console to show the scale of the service catalog — AWS has grown from 5,000 services a few years ago to an even larger catalog today. AWS pushed hard into universities with AWS Academy, which mirrors what Microsoft and Google do: get students familiar with your environment early, because today's students are tomorrow's IT buyers.

EC2 Instance Types

  • Each instance type has a name, architecture, vCPU count, memory, storage, and network performance profile.
  • Pricing is shown per hour and varies by OS; Linux is consistently cheaper than Windows in server workloads.
  • AWS Academy student accounts are restricted to the us-east-1 region because it is the cheapest data center.
  • Data residency rules often force enterprises to stay within specific regions (EU, US, etc.).
Edge Computing

Even inside a single cloud provider, choosing where to run a job is a real optimization problem. Edge computing is about reducing the distance data has to travel, cutting latency, improving real-time responsiveness, and lowering bandwidth costs.

S3 Storage Tiers and Cost Structure

Amazon S3 is AWS's core object storage. Data is organized into buckets, and the economics drive major architectural decisions.

Tier Approximate Cost per GB / Month Best For
S3 Standard ~$0.023 Frequently accessed hot data
S3 Infrequent Access ~$0.01 Data that is read occasionally
S3 Glacier Fraction of a cent Archival data accessed rarely
S3 Glacier Deep Archive ~$0.001 or less Records accessed once or twice a year; retrieval can take ~12 hours
Scale Numbers

S3 can hold effectively unlimited total data (up to exabyte scale). Individual objects range from 0 bytes to 5 TB. An AWS account can create up to 1 million buckets. Most teams use many buckets to organize data, manage permissions, and scale around per-bucket request limits.

The broader point: access pattern determines tier, and tier determines cost. If you have historical records only pulled for rare FOIA requests, archival tiers are dramatically cheaper. If you need millisecond access, you pay Standard pricing.

Other Service Categories and Clusters on Demand

Beyond compute and storage, Dr. Pierce briefly surveyed the breadth of the AWS catalog: databases, migration and transfer, networking and content delivery, developer tools, blockchain, satellite, quantum, management and governance, media, game development, IoT, end-user computing, business applications, application integration, front-end mobile, cloud financial management, security and compliance, analytics, and machine learning.

AWS EMR (Elastic MapReduce) was previewed as a preview of next class's topic. Creating a Hadoop / Spark / Flink / HBase / Presto cluster used to mean buying hardware, racking it, wiring the network, installing the OS, installing the cluster software, and testing — now it is a menu of instance types, a security key setup, and a "Create Cluster" button. Provisioning takes about 20 minutes.

The EC2 Key Pair as "Secret Handshake"

Dr. Pierce compared the EC2 key pair to the secret handshake at a childhood clubhouse. The cluster will not accept connections from a machine that does not present the matching key, which is how access is gated.
