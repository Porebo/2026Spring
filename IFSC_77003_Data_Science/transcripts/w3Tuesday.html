🐍 Why Python for Data Science?

Python has become one of the top 10 most popular languages due to its unique philosophy:

  • Concise conversion: Minimizes curly braces and semicolons to focus on functionality.
  • Interpreted System: Allows for rapid "prototyping" and testing without waiting for long compile times.
  • Data-Centric: Built to handle big data, mathematical operations, and complex statistical analysis through powerful libraries.

📏 Key Syntax Rules

Python differs from many C-based languages in critical ways:

  • Indentation Matters: Whitespace is not just for looks; it defines command blocks (e.g., inside an if statement). Incorrect spacing causes errors.
  • Duck Typing: You don't need to declare if a variable is an Integer or String; Python "infers" the type based on the value assigned.
  • Comments: Use # for single lines or ''' triple quotes for multi-line documentation.

⚖️ Scope: Global vs. Local

Understanding where data "lives":

  • Global: Valid throughout the entire program.
  • Local: Restricted to a specific part (like a function).
  • The "Global" Keyword: You can force a variable created inside a function to be global so the rest of the program can see it.

📦 Python Data Types

Standard types used in analysis:

  • Numeric: int, float, and complex numbers (using `j`).
  • Collections: Lists, Tuples, Dictionaries, and Sets.
  • The "None" Type: Used to represent the absence of a value (similar to Null).
