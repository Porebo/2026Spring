CNN Image Recognition, Training Workflow, and Overfitting Control

March 5, 2026 Dr. Elizabeth Pierce UALR
What CNNs Do Layer Mechanics Data Preparation Training and Epochs Evaluation Project Direction

This class transitioned from basic image preprocessing into predictive image modeling, showing how convolutional neural networks can classify handwritten digits when supported by disciplined data preparation, tuning, and evaluation.

From Processed Images to Prediction

Dr. Pierce framed CNNs as specialized deep-learning models for grid-like data, with image pixels as the canonical example. The session used a digit-recognition challenge to make architecture and training concepts concrete.

Practical Framing

The instructional goal shifted from image cleanup (grayscale, blur, threshold) toward predicting labels on new unseen images.

CNN Layer Roles

  • Convolution layers: extract local patterns such as strokes and edges.
  • Pooling layers: reduce dimensionality and improve robustness to small shifts/variations.
  • Dense/output layers: combine extracted features to produce class probabilities.

Data Preparation in the Demo

The notebook walkthrough showed the required preprocessing path for handwritten-digit classification: split labels/features, inspect class balance, normalize pixel intensities from 0-255 to 0-1, reshape into tensors, and perform train/test partitioning.

  • Dataset context: digit labels with 28x28 pixel values per sample.
  • Train/test setup: example 80/20 split with configurable randomization.
  • Label encoding: prepared target values for network training.

Training Logic, Epochs, and Overfitting

The lecture explained iterative weight updates and why epoch count needs calibration. Too few epochs can underfit; too many can overfit by memorizing training details.

Overfitting Risk

High performance on training data does not guarantee generalization. Validation behavior and unseen-data tests are essential before trusting deployment decisions.

Mitigation techniques discussed included dropout, pooling choices, and data augmentation (rotations/shifts/transformations) to increase robustness against variation.

A good CNN is not just accurate on known samples; it remains reliable on new handwriting it has never seen.

Model Evaluation Principle

Diagnostic Evaluation

  • Validation-loss curves were used to locate useful epoch ranges and detect degradation patterns.
  • Confusion matrices highlighted class-specific mistakes (for example, where similar digits are confused).
  • Iterative tuning was demonstrated by adjusting configuration and comparing diagnostic outcomes.

Project and Career Implications

Students were encouraged to apply the workflow to project ideas in Unit 12, especially where image/video recognition intersects with moderation, multimodal fusion, synthetic data, or real-time decision support.

  • Review Unit 6/7 demos and adapt notebook pipelines for project scoping.
  • Document tradeoffs among model accuracy, interpretability, compute cost, and ethical risk.
  • Consider cloud student/research credits when computational resources are a bottleneck.
