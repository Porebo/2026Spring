## 3. Project Scope and Effort
- Target report length: 10-20 pages (graduate baseline; undergrad follows same structure with grading curved accordingly)
- Formatting guidance: 12-point font, single spacing (about 1.15), 1-inch margins
- Typical effort for a top-rated project: 40-60 hours

## 4. Individual vs Coordinated Projects
- Each student must complete their own project deliverables and is graded individually.
- Coordinated projects are allowed (students can work on different parts of a broader problem), but each person must independently provide:
  - substantial analysis work,
  - their own report,
  - their own 3-minute presentation,
  - their own documented effort.

## 5. Required Report Structure (Parts 1-10)

### Part 1: Title Page / Title Section
Include:
- [ ] Project title
- [ ] Your name
- [ ] Supervisor name
- [ ] Institution and department
- [ ] Course number and name
- [ ] Semester/date

### Part 2: Table of Contents
- Optional for this class (okay to omit).

### Part 3: Introduction
Cover:
- [ ] Topic and motivation
- [ ] Background/context (what is known, what is missing)
- [ ] Problem statement and research questions
- [ ] Why the project is worth doing

### Part 4: Literature Review
- [ ] 10-15 sources included
- [ ] Compare/contrast theories, methods, and debates
- [ ] Explain how your project builds on/challenges prior work
- [ ] Verify that referenced sources actually exist (important when using AI tools)

### Part 5: Data Selection, Research Design, and Methods
Include:
- [ ] Restated objectives
- [ ] Overall strategy for answering research questions
- [ ] Data sources and data quality notes
- [ ] Data science techniques used for analysis

### Part 6: Analysis Work
- [ ] Present and interpret your analysis results.

### Part 7: Discussion of Results
Include:
- [ ] How results answer project objectives/questions
- [ ] Security, privacy, and ethical considerations for real-world deployment
- [ ] Disclosure of whether/how AI was used in your project

### Part 8: Conclusions and Closing Thoughts
Include:
- [ ] Summary of findings
- [ ] Key contributions/insights
- [ ] Lessons learned
- [ ] Future work ideas

### Part 9: Appendix
Include:
- [ ] Project schedule/timeline
- [ ] Hours spent by project stage
- [ ] Reference list (full citations)
- [ ] Optional supporting documentation (code/notebook links, extra materials)

### Part 10: 3-Minute Thesis File/Link
- [ ] Provide a file or link to your 3-minute presentation.

## 6. 3-Minute Thesis (3MT) Constraints
- Maximum speaking time: 3:00 (longer talks may not be viewed / can be disqualified)
- Audience: non-specialist (clear, simple language)
- You may use one single static PowerPoint slide only
- Do NOT use transitions, animations, embedded audio/video, or other media
- Do NOT use props
- Spoken-word presentation

Strong talks typically include:
- [ ] Background/significance
- [ ] Method/approach
- [ ] Key findings
- [ ] Conclusions/impact
