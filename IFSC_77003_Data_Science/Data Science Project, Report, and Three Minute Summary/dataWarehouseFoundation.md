Enterprise dashboards and reporting tools do not operate in isolation, but depend on a broader data strategy that governs how raw data is extracted, transformed, centralized, and made available for value creation. Balakrishnan et al. (2020) explain that organizations create data-enabled value through coordinated stages such as data extraction, data transformation, value creation, and value delivery, supported by appropriate data management architecture and governance. This perspective is important because it shows that analytical tools derive their usefulness from curated data environments rather than from visualization software alone. In other words, effective dashboards depend on an underlying enterprise system that turns distributed operational data into organized and analyzable information (Balakrishnan et al., 2020).

In the context of this project, Balakrishnan et al. (2020) support the description of Aera Energy's Oracle Data Warehouse (ODW) as the central analytical foundation for later dashboard use. Operational data from multiple source applications was extracted, transformed, and centralized in ODW, making curated data available for analysis and decision support. In this environment, Tableau reports were enabled by a broader enterprise data strategy rather than standing alone as isolated visual tools. ODW served as the repository that made it possible to combine information from multiple operational systems into a form usable by technicians, analysts, engineers, and managers through Excel extracts and later through Tableau dashboards. This helps frame the Tableau-to-Spotfire problem not only as a reporting issue, but as one layer of a larger enterprise data-to-value process built on curated warehouse data (Balakrishnan et al., 2020).
